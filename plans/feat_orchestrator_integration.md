# Feature Plan: Orchestrator Integration

## Objective
Integrate the existing `ochestrator` (tmux pane summarizer) into Workstation Lite. This will provide users with a clean UI dialog to view the real-time status and task summaries of all AI agents (running in `tmux` panes) directly from the Workstation Lite dashboard. The orchestration API runs inside the workstation pod and uses the Gemini CLI to summarize pane activity.

## Key Files & Context
- **Workstation Image:** `templates/Dockerfile.template` and a new `run.sh` entrypoint.
- **Backend Routes:** `backend/app/api/workstations.py`
- **Backend Services:** `backend/app/services/k8s.py`
- **Frontend Components:** `frontend/src/App.tsx`, `frontend/src/components/AgentStatusDialog.tsx`
- **Orchestrator Source:** `../ochestrator/tmux-api/main.py` and `../ochestrator/tmux-summarizer/scripts/get_tmux_context.sh`

## Implementation Steps

### 1. Workstation Image Configuration (The "Agent Orchestrator" Server)
We will bake the `tmux-api` directly into the base Workstation Docker image so it is available out of the box.
- **Modify `templates/Dockerfile.template`**:
  - Install dependencies: `python3`, `pip`, `fastapi`, `uvicorn`, `tmux`, and the `gemini` CLI.
  - Create a directory `/opt/orchestrator`.
  - Copy `../ochestrator/tmux-api/main.py` and `../ochestrator/tmux-summarizer/scripts/get_tmux_context.sh` into the image.
  - Use `sed` or patch `main.py` inside the Dockerfile to update the hardcoded `TMUX_SCRIPT_PATH` to point to `/opt/orchestrator/get_tmux_context.sh`.
- **Modify Entrypoint (`run.sh`)**:
  - Create or inject a `run.sh` script that starts `uvicorn main:app --host 0.0.0.0 --port 8001 &` in the background from `/opt/orchestrator` before starting the primary IDE process (e.g., OpenVSCode Server).
  - Ensure the container user has proper authentication (ADC or API keys) mapped so the Gemini CLI can function.

### 2. Backend Proxy API
The frontend cannot reach port `8001` on the workstation directly, so the Python backend will proxy the requests using the native Kubernetes pod proxy API.
- **Add Service Method** in `backend/app/services/k8s.py`:
  - `get_workstation_agents(self, user_ns: str, name: str) -> dict`
  - Use `self.core_api.connect_get_namespaced_pod_proxy_with_path(f"{name}-0", user_ns, path="api/panes", port="8001")`.
  - Handle exceptions gracefully (e.g., 504 Timeouts if the server isn't up).
- **Add API Endpoint** in `backend/app/api/workstations.py`:
  - `GET /{user_ns}/agents/{name}`
  - Calls `get_workstation_agents` and returns the JSON payload containing the `panes` array.

### 3. Frontend UI/UX
Create an interactive dialog for users to review agent activity.
- **Trigger Button**: In `App.tsx` (or `WorkstationCard`/row component), add a new "Agents" icon button (e.g., a bot or terminal icon) next to the Start/Stop controls for running workstations.
- **Component `AgentStatusDialog.tsx`**:
  - A Material-UI `Dialog` that opens when the button is clicked.
  - **Loading State**: Displays a `CircularProgress` and the text "Querying agent states... This takes a few seconds to run the AI summarization." while the proxy request is inflight.
  - **Data View**: Maps over the returned `panes` array and renders a clean list of cards or a structured table:
    - **Header**: Displays `window_name` and `command`.
    - **Status Badge**: A colored chip (Green for `RUNNING`, Gray/Yellow for `WAITING`).
    - **Task Summary**: A Typography block containing the plain text or markdown task description generated by Gemini.
  - **Refresh Action**: An `IconButton` in the dialog title bar to re-trigger the API fetch without closing the dialog.
  - **Error Handling**: An Alert banner displaying "Failed to communicate with Agent API." if the proxy call fails or times out.

## Verification & Testing
1. **Build a Workstation**: Rebuild the workstation image containing the new orchestrator server. Ensure the container starts both `uvicorn` on 8001 and the main IDE process successfully without crashing.
2. **Tmux Setup**: Start a few named `tmux` sessions with generic processes (e.g., a dummy `python -m http.server` loop) inside the workstation.
3. **API Integrity**: Execute the Workstation Lite backend endpoint via `curl` or Swagger UI and verify it successfully proxies to the pod's port 8001 and returns valid JSON.
4. **UI Validation**: Open the Workstation Lite UI, click the "Agents" button, and verify the dialog correctly displays the summarizing loading state, handles errors, and presents the parsed pane data clearly. Verify the "Refresh" button accurately updates the data.
